Movie API

The objective of this project is to build a server-side component of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favourite movies.

Endpoints

The table below documents the endpoints in this API - including details of what URL endpoints to target, the data to send as parameters and the data to expect as responses.

Business Logic URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of ALL movies to the user /movies GET None A JSON array holding data about all movies in database
Return data on single movie by title /movies/:title GET None A JSON object holding data about a single movie, containing movie title, description, genre details, director details, image URL and if featured. Example:
{
"Genre": {
"Name": "Drama",
"Description": "Drama films are serious presentations or stories with settings or life situations that portray realistic characters in conflict."
},
"Director": {
"Name": "David Fincher",
"Bio": "David Andrew Leo Fincher is an American film director.",
"Birth": "1962"
},
"_id": "668d2c4d22f883aa38b81b3b",
"Title": "Fight Club",
"Description": "An insomniac office worker and a devil-may-care soap maker form an underground fight club that evolves into much more.",
"ImagePath": "fightclub.png",
"Featured": false
}
Return data on genre by name /movies/Genre/:GenreName GET None A JSON object holding data about a movie genre, including name and description:
{
"Name": "Drama",
"Description": "In film and television, drama is a category of narrative fiction (or semi-fiction) intended to be more serious than humorous in tone."
}
Return data on director by name /movies/Director/:DirectorName GET None A JSON object holding data about a director, containing name, bio and birth year:
{
"Name": "Steven Spielberg",
"Bio": "American film director, writer and producer. A major figure of the New Hollywood era and pioneer of the modern blockbuster, he is the most commercially successful director of all time. Spielberg is the recipient of various accolades, including three Academy Awards, a Kennedy Center honor, four Directors Guild of America Awards, two BAFTA Awards, a Cecil B. DeMille Award and an AFI Life Achievement Award.",
"Birth": "1946"
}
Add a new user /users POST A JSON object holding data about the user to add, structured like:
{
"Username": "newUser",
"Password": "1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05",
"FavoriteMovies": []
} 		A JSON object holding data about the user that was added, including an ID:
{
"Username": "newUser",
"Password": "1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05",
"FavoriteMovies": []
"_id": "66964b7f1c03061b40878417",
"__v": 0
}
Update user info /users/:Username PUT A JSON object holding data about the user to update, structured like:
{
"Username": "newUser",
"Password": "updated1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05",
"FavoriteMovies": []
} 		A JSON object holding data added to the user:
{
"_id": "669644b36667eff5997cc420",
"Username": "newUser",
"Password": "updated1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05T00:00:00.000Z",
"FavoriteMovies": [],
"__v": 0
}
Add a movie to list of favorites /users/:Username/movies/:MovieID POST None A JSON object holding data of user including added favorite movie:
{
"_id": "669644b36667eff5997cc420",
"Username": "newUser",
"Password": "updated1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05T00:00:00.000Z",
"FavoriteMovies": [
"668d2bad22f883aa38b81b39"
],
"__v": 0
}
Remove a movie from list of favorites /users/:Username/movies/:MovieID DELETE None A JSON object holding data of user minus deleted movie:
{
"_id": "66964b7f1c03061b40878417",
"Username": "newUser",
"Password": "1234",
"Email": "new1234@gmail.com",
"Birthday": "1990-05-05T00:00:00.000Z",
"FavoriteMovies": [],
"__v": 0
}
Deregister existing user /users/:Username DELETE None A text message indicating whether the user was successfully deleted
